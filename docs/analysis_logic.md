# TradeAlgo Pro 判定ロジック詳細

本アプリの分析ロジックは複数の独立したレイヤーで構成されており、選択したトレードスタイル（長期・スイング・デイトレ）に合わせて各レイヤーの重み付けや配点が自動調整されます。

## 0. 基本構成と統合スコア (Integrated Score)

分析の中核は **Layer 3 (テクニカル)** と **Layer 6 (先回り検知)** です。これらを統合したスコアが最終的な売買判断（Signal）の基礎となります。

### 統合スコア計算式 (動的正規化)
`Total Σ = Σ(各Layerのスコア × 重み) / Σ(各Layerの有効な最大点数 × 重み)`

データの取得状況（APIキーの有無、データの欠損、ニュースの有無など）に応じて、**データが入手できた項目のみ**で100%換算されるよう、各レイヤーの最大点数（分母）が自動調整されます。

- **厳格なデータ判定**: 
  - `0.0` (実績としてのゼロ) は評価対象として分母に残ります。
  - `None` または `NaN` (取得不可) は評価不能として分母から除外されます。

| トレードスタイル | L6 (需給) | L3 (テク) | L2 (財務) | L4 (定性) | L5 (配当) |
| :--- | :---: | :---: | :---: | :---: | :---: |
| **長期保有 (配当)** | **1.56** (x1.3) | **0.70** (x0.7) | **1.30** (x1.3) | **1.00** | **1.30** (x1.3) |
| **スイング (標準)** | **1.20** | **1.00** | **1.00** | **1.00** | (除外) |
| **デイトレ (短期)** | **0.84** (x0.7) | **1.30** (x1.3) | (除外) | **1.00** | (除外) |

*注：各レイヤー内部の特定項目（例：L2のROEデータのみ欠如）についても、その配点分がレイヤーの最大点数から動的に差し引かれます。*

## 1. 分析レイヤー詳細

### Layer 1: マクロ・リスクフィルター
市場全体の環境を分析し、個別銘柄のスコアに補正を加えます。
- **補正ロジック**:
  - **グロース株減点**: 米10年金利 > 4.5% または NASDAQが75日線下の場合、PER>30の銘柄を各5点減点。
  - **円安メリット加点**: USD/JPYが2%以上急騰した場合、日本株を5点加点。
- **ブロック（ストッパー）**:
  - **VIX 40超**: 「パニックモード」として全ての新規買い判定をブロック。
  - **VIX 30〜40**: 「高リスク / 警戒モード」。警告ラベルを付与。

### Layer 2: ファンダメンタル
- **配点**: 最大40点（Growth 20点 / Valuation 20点 [PER:7, PBR:6, ROE:7]）。
- **デイトレモード**: 分析から**完全に除外**されます。

### Layer 3: テクニカル
- **指標**: EMA(9/20/50/75/200), RSI, MACD, RVOL, ボリンジャーバンド。
- **出来高 (RVOL) 共通配点**:
  - RVOL ≥ 2.0: **+10点**（強い資金流入）
  - RVOL ≥ 1.5: **+6点**（トレンド発生）
  - RVOL ≥ 1.3: **+3点**（初動検知）
- **スタイル別判定**:
  - **長期 (Long)**: 200日EMA基準。傾き判定は 25日間。
  - **スイング (Swing)**: 50日EMA基準。EMA20との位置関係、EMA20反発、EMA50傾き（15日間）を重視。
  - **デイトレ (Day)**: 9/20/50日EMA基準。パーフェクトオーダー、EMA9反発、RVOL高騰を重視。

### Layer 4: 定性・ニュース（AI判定）
- **AI**: Gemini AI による感情分析。スタイルにより配点と解析視点（短期・配当等）が変動。
- **ボーナス**: 24時間以内のニュース件数急増時に最大+4点のトレンド加点。

### Layer 5: 配当・インカム（長期保有限定）
- **評価項目**: 利回り（平均比較）、配当性向（20-60%が理想）、グレアム指数。

### Layer 6: 先回り検知・アキュムレーション (Accumulation)
「これから動く」兆候を検知する独立レイヤー（最大40点）。
- **主要評価**: ダイバージェンス、セクター乖離、ボラティリティ収縮（スクイーズ）、静かな出来高増、EMA接近、割安成長。
- **信頼性**: ヒット項目数（1-6）に基づき「High / Normal / Low」で格付け。3未満は「信頼性不足」として減点。

## 2. シグナル判定の制約 (Constraints)

単純な合計点だけでなく、以下の「質のチェック」を経て最終シグナルが決定されます。

### 減衰 (Attenuation)
- **L6不足制限**: Layer 6 < 15点の場合、テクニカル(L3)評価に **0.85倍の減衰係数** を適用。
- **長期トレンド減衰**: 長期モードで価格 < EMA200 の場合、L3評価を **0.75倍** に減衰（ストッパー非発動時）。
- **スイングトレンド減衰**: スイングモードで価格 < EMA200 の場合、L3評価を **0.7倍** に減衰。
- **デイトレレンジ減衰**: 過去10本でEMA9とEMA20が3回以上交差する場合、L3評価を **0.8倍** に減衰。

### 判定の引き下げ・禁止
- **Strong Buy 禁止**: L6 < 20 の場合、または L3スコアが前日比で増加していない（勢いがない）場合。
- **RSI過熱**: 70以上の場合は、判定を1段階引き下げ。
- **ダブル・ドロップ**: L3とL6の両方が前日比で低下している場合、Sell/Avoidを優先。

### 判定の引き上げ
- **仕込み継続**: L6 >= 25 かつ L3 < 15 の場合、低迷期でも「Buy（仕込み）」として評価。
- **セクター流合**: Layer 0（セクターローテーション）で強いセクターと一致する場合、1段階引き上げ。

## 3. ストッパーと緩和 (Stoppers & Exceptions)
以下の条件に該当する場合、通常は評価を無効化し **「Sell / Watch (ストッパー発動)」** としますが、特定の需給条件（L6/RVOL）を満たす場合は「例外的に許可（仕込み推奨）」となります。

### ストッパー条件
1. **長期下降トレンド**: EMA200の傾き < 0 かつ 価格 < EMA200。
2. **スイング下降トレンド**: EMA200の傾き < 0 かつ 価格 < EMA50。
3. **決算リスク**: 決算発表の5日前から当日まで。
4. **急落**: 直近60日高値から20%以上の下落（長期モードでは L6 < 20 の場合のみ発動）。
5. **大幅窓開け**: 前日比±5%以上の乖離発生。

### 例外解放（仕込みOK）
- **長期**: L6 ≥ 25 かつ RVOL ≥ 1.3 かつ（ボラ収縮 または ダイバージェンス）
- **スイング**: L6 ≥ 25 かつ RVOL ≥ 1.2

## 5. 銘柄検索・同定ロジック (Search & Identification)

ユーザーが入力したキーワードから適切な銘柄を特定するための、入力支援・優先順位付けロジックです。

### 日本市場優先表示
グローバルな検索結果の中から、以下の条件に基づいて日本市場の銘柄を自動的に識別し、リストの最上位にソートします。
- **サフィックス判定**: シンボルが `.T` (東京), `.O` (大阪), `.S` (札幌) 等で終わる場合。
- **取引所名判定**: 取引所名に `Tokyo`, `Osaka`, `Nagoya`, `Sapporo`, `Fukuoka` などの国内主要市場の名称、または `JPX` を含む場合。

### 特定パターン補完
- **4桁数字入力**: 入力値が4桁の数字のみの場合、検索結果に未反映であっても `.T` (東証) を付与した候補を最優先で生成し表示します。
- **検索ヒント**: 日本語名での検索など、結果が0件となる可能性が高い入力を検知した際、英語名やコードでの入力を促す自動ヒント（NOTICE）を表示します。
