# TradeAlgo Pro フロントエンド仕様書

このドキュメントでは、TradeAlgo Pro（株式自動判定ツール）のフロントエンド側（React）のディレクトリ構成、および各プログラムファイルの役割について解説します。

## 📁 フロントエンド ディレクトリ構成

```text
src/
├── App.tsx             # メインアプリケーションコンポーネント (UI・ロジック統括)
├── SettingsModal.tsx   # APIキー設定ダイアログコンポーネント
├── types.ts            # TypeScript型定義 (バックエンドモデルと同期)
├── App.css             # グローバルおよびコンポーネント用スタイルシート
├── index.css           # リセットCSSおよび基本変数
├── main.tsx            # Reactアプリケーションの起動・マウントポイント
└── vite-env.d.ts       # Vite用環境変数や型定義
```

## 📄 各ファイルの詳細説明

### 1. `App.tsx`
**役割：メインのUI表示とアプリケーションの状態管理**
* ユーザーインターフェースの中核であり、以下の機能を持ちます：
  * **入力フォーム**: 銘柄コード、時間軸（1分足〜週足）、トレードスタイルを入力し、バックエンドへ送信。
  * **状態管理**: 分析中、取得結果、エラー情報、接続状況などのStateを管理。
  * **結果の表示**: バックエンドから返ってきた `AnalysisResult` を可視化します。
    * 総合シグナル、マクロバッジ（安全/警戒/停止）、レイヤー別スコア。
    * **テクニカルメトリクス**: 5/20/75日EMA、RSI、MACD、VWAP、ボリンジャーバンド、出来高比の実数値をカード形式で表示。
    * **セクター情報**: 資金流入/流出セクターの表示。
    * **リスク管理**: 推奨損切りラインやボラティリティ警告の表示。
  * **チャート機能**: `recharts` を使用。判定結果に応じた色（Buyなら緑、Sellなら赤）がチャートに同期。
  * **共有機能 (レポート生成)**:
    * **Web Share API**: モバイルブラウザのネイティブ共有機能を使用。
    * **多段フォールバック**: 1. 画像＋テキスト共有 → 2. テキストのみ共有 → 3. クリップボードコピー の順に試行し、HTTP環境や古いブラウザでも確実に結果を保持可能。
    * **html2canvas**: チャート領域をキャプチャし、共有用画像として利用。

### 2. `SettingsModal.tsx`
**役割：ユーザー固有の設定（APIキー）を管理するモーダル**
* **J-Quants API連携用リフレッシュトークン** と **Google Gemini APIキー** を入力。
* 情報はブラウザの `localStorage` に保存。

### 3. `types.ts`
**役割：TypeScriptの型・インターフェース定義**
* バックエンドの `models.py` と同期された最新の型定義。
* `TechnicalResult` には `macd`, `vwap`, `bollinger_upper/lower`, `ema5/20/75` 等が含まれます。
* `MacroResult` には為替トレンド、金利、セクターローテーション等の市場データ構造が定義されています。

### 4. `App.css`
**役割：スタイリングコア**
* アプリケーション全体の美しいダークテーマとグラスモーフィズムデザインを担っています。

### 5. `main.tsx` / `index.css`
**役割：起動処理と全体の基本設定**
* `main.tsx`: Reactツリーの起点。
* `index.css`: アプリ全体のカラーパレット、フォント設定。
