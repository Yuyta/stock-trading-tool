# TradeAlgo Pro フロントエンド仕様書

このドキュメントでは、TradeAlgo Pro（株式自動判定ツール）のフロントエンド側（React）のディレクトリ構成、および各プログラムファイルの役割について解説します。

## 📁 フロントエンド ディレクトリ構成

```text
src/
├── App.tsx             # メインアプリケーションコンポーネント (UI・ロジック統括)
├── SettingsModal.tsx   # APIキー設定ダイアログコンポーネント
├── types.ts            # TypeScript型定義 (バックエンドモデルと同期)
├── App.css             # グローバルおよびコンポーネント用スタイルシート
├── index.css           # リセットCSSおよび基本変数
├── main.tsx            # Reactアプリケーションの起動・マウントポイント
└── vite-env.d.ts       # Vite用環境変数や型定義
```

## 📄 各ファイルの詳細説明

### 1. `App.tsx`
**役割：メインのUI表示とアプリケーションの状態管理**
* ユーザーインターフェースの中核であり、以下の機能を持ちます：
  * **入力フォーム**: 銘柄コード、時間軸（1分足〜週足）、トレードスタイル（スイング / デイトレ）を入力し、バックエンドへ送信します。
  * **状態管理**: 分析中（`isAnalyzing`）、取得結果（`result`）、エラー情報（`error`）、バックエンドとの接続状況（`backendOnline`）などのStateを管理します。
  * **通信処理**: Fetch API を用いて `POST /api/analyze` 呼び出しを行います。
  * **結果の表示**: バックエンドから返ってきた `AnalysisResult` オブジェクトをパースし、以下を表現します。
    * 総合シグナル（Strong Buy, Sell など）、対応するアイコン、現在のモード名
    * レイヤー別の表示（マクロバッジ、ファンダメンタル、テクニカル、定性・ニュース スコア）
    * リスク管理指標（損切りライン等）
  * **チャート機能**: `recharts` ライブラリを使用して `chart_data` を元に美麗な折れ線（Area）チャートを描画します（X軸に時刻/日付、Y軸に株価）。

### 2. `SettingsModal.tsx`
**役割：ユーザー固有の設定（APIキー）を管理するモーダル**
* **J-Quants API連携用リフレッシュトークン** と **Google Gemini APIキー** をユーザーが入力するための画面コンポーネントです。
* セキュリティと利便性を考慮し、入力された情報はサーバーには永続保存せず、ブラウザの `localStorage`（ローカルストレージ）に保存・読み込みを行います。
* 「保存」ボタンが押されると、親コンポーネント（`App.tsx`）に設定情報が引き渡され、以降のAPIリクエストに添付されるようになります。

### 3. `types.ts`
**役割：TypeScriptの型・インターフェース定義**
* フロントエンド全体で利用されるデータ構造を定義します。
* バックエンド側（Python/FastAPI）の `models.py` で定義されているPydanticモデルと**完全に1対1**で対応（同期）するように作られています。
* これにより、API通信時のデータ・プロパティ名（`score`, `max_score`, `data_source` 等）でタイポや型の不一致が発生するのを未然に防ぎます。

### 4. `App.css`
**役割：スタイリングコア**
* アプリケーション全体の美しいダークテーマとグラスモーフィズムデザインを担っています。
* 主要なクラス：
  * `.glass-panel`: 背景を透過させ、磨りガラスのような効果を付与。
  * `.score-bar-track` / `.score-bar-fill`: 点数を示すバーのアニメーション・デザイン。
  * `.layer-status` / `.alert-box`: 成功、警告、エラーなどのステータス表示用バッジデザイン。
  * `@media (max-width: 768px)`: モバイル（スマホ）での閲覧時にレイアウトを1カラムにし、余白やフォントサイズを調整するためのレスポンシブデザイン定義。

### 5. `main.tsx` / `index.css`
**役割：起動処理と全体の基本設定**
* `main.tsx`: Reactツリーの起点。`App` コンポーネントをDOM（`index.html`の`#root`）に描画（レンダリング）します。
* `index.css`: アプリ全体のカラーパレット（CSS変数）、フォント設定、マージンのリセットなど、全ページ共通のベーススタイルを定義しています。

## ⚙️ 環境設定とビルド (ルート直下のファイル)
* **`vite.config.ts`**: Viteの開発サーバー設定。`/api` へのリクエストをバックエンド（ポート8000）へ転送（プロキシ）する設定や、モバイル検証用にサーバーを同一ネットワーク内に公開（`host: true`）する設定を含みます。
* **`package.json`**: React, Vite, Lucide React (アイコン), Recharts (チャート) など、フロントエンドで利用しているnpmパッケージのリストと起動コマンドが定義されています。
