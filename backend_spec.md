# TradeAlgo Pro バックエンド仕様書

このドキュメントでは、TradeAlgo Pro（株式自動判定ツール）のバックエンド側のディレクトリ構成、および各プログラムファイルの役割について解説します。

## 📁 バックエンド ディレクトリ構成

```text
backend/
├── main.py            # アプリケーションのエントリーポイント (FastAPI)
├── models.py          # データモデル定義 (Pydantic リクエスト/レスポンス型)
├── data_fetcher.py    # 外部APIからのデータ収集モジュール
├── analyzer.py        # 判定アルゴリズム・スコアリングロジックコア
├── requirements.txt   # Python パッケージ依存関係
└── start.bat          # 開発時起動用バッチスクリプト
```

## 📄 各ファイルの詳細説明

### 1. `main.py`
**役割：APIサーバーのエントリーポイント**
* FastAPI フレームワークを使用してWebサーバーを立ち上げます。
* CORS（Cross-Origin Resource Sharing）の設定を行い、フロントエンド（React等）からのAPIリクエストを受け付けるようにしています。
* `/api/health`: サーバーの稼働状態を確認するためのヘルスチェック用エンドポイントです。
* `/api/analyze`: フロントエンドから銘柄コードや条件（時間軸、トレードスタイルなど）を受け取り、`analyzer.py` の分析処理を呼び出し、結果をフロントエンドに返します。

### 2. `models.py`
**役割：データ構造の定義層**
* `pydantic.BaseModel` を活用し、フロントエンドとやり取りするJSONデータの型（スキーマ）を定義しています。
* **リクエストモデル**: `AnalyzeRequest`（入力される銘柄コード、時間軸、トレードスタイル、APIキー等）。
* **レスポンスモデル**: マクロ(`MacroResult`)、ファンダメンタル(`FundamentalResult`)、テクニカル(`TechnicalResult`)、定性・ニュース(`QualitativeResult`)、リスク情報(`RiskInfo`)、チャート表示用(`ChartDataPoint`)、そしてそれらを統括する `AnalysisResult` メインクラス。

### 3. `data_fetcher.py`
**役割：外部データソースとの通信を担うモジュール**
* 他のモジュールから独立して、株価や業績データを外部から取得します。
* **yfinance**: 米国株および日本株（`.T` 付与）の価格履歴、出来高、単純なファンダメンタルデータ、英語・日本語のニュースヘッドラインを取得します。
  * `fetch_price_history`: 分足（デイトレ用）〜日足・週足（スイング用）まで、引数に応じた期間・インターバルでチャートデータを取得します。
* **J-Quants API**: 日本株の正確な財務データ（PER、PBR、ROE、四半期業績等）を取得します。ユーザーがリフレッシュトークンを提供した場合のみ作動します。

### 4. `analyzer.py`
**役割：コア分析アルゴリズムとスコアリング**
* 取得したデータを元に、四層フィルターを通して銘柄の魅力やリスクを算出します。
* 引数・設定により**フルモード**（Gemini/J-Quantsあり）、**標準モード**（どちらかあり）、**基本モード**（APIキーなし）、**デイトレモード** のスコア配分や計算方式をダイナミックに変化させます。
* **1. マクロ環境フィルタ (`_analyze_macro`)**: VIX指数や金・原油価格、日経225等を監視し、相場全体がクラッシュ寸前（VIX>30など）の場合は個別株のスコアに関わらず「買い見送り」の強制ブロックを発動します。
* **2. テクニカル分析 (`_analyze_technical`)**: トレンド（EMAのゴールデンクロス・位置関係）、モメンタム（RSI）、出来高の急増などを数学的に判定します。
* **3. ファンダメンタル分析 (`_score_fundamental`)**: PER/PBRや利益成長率を分析します（デイトレモード時は除外されます）。
* **4. 定性・ニュース分析 (`_score_qualitative`)**: Gemini APIが設定されていればプロンプトを投げてAIが10点満点で採点。未設定の場合はキーワードの出現回数をカウントして簡易的に最大7点で採点します。
* これらを統合した最終スコアと最大スコア（max_score）を割り出し、「Strong Buy / Buy / Hold / Sell / Avoid」のシグナルを生成します。

### 5. `requirements.txt`
* APIサーバー (`fastapi`, `uvicorn`)
* データ処理・解析 (`pandas`, `numpy`)
* 外部連携 (`yfinance`, `requests`, `google-generativeai`)
などのバージョンをロック・明記しているファイルです。

### 6. `start.bat`
* Windows 開発環境向けに、Python仮想環境の作成から依存ライブラリのインストール、Uvicornサーバーの起動 (`uvicorn main:app --reload --port 8000`) までを自動で行うユーティリティスクリプトです。
