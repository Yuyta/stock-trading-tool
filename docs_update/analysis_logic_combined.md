# TradeAlgo Pro 判定ロジック詳細

本アプリの分析ロジックは4つの独立したレイヤーで構成されており、状況に合わせてモードや配点が自動調整されます。

# Layer 0: セクターローテーション分析（追加）

市場では常に **資金がセクター間を移動（ローテーション）**しています。  
個別銘柄より先に **どのセクターに資金が流入しているか** を分析することで  
銘柄選定の精度を向上させます。

## 監視対象

主要セクター指数またはETF

例

- 半導体
- AI / ソフトウェア
- 自動車
- 銀行
- 商社
- エネルギー
- 防衛
- 小売
- バイオ

米国市場参考セクター

- XLK（テック）
- XLF（金融）
- XLE（エネルギー）
- XLY（一般消費）

## 判定ロジック

### 相対強度（Relative Strength）

各セクターのパフォーマンスを  
市場指数と比較します。

## 1. 分析レイヤー概要

### Layer 1: マクロ・リスクフィルター
- **監視指標**:  
  VIX（恐怖指数）、WTI原油、金価格、市場平均（日経225）。  
  **（追加）米10年金利（US10Y）、USD/JPY為替、NASDAQ指数**

- **判定ロジック**:
  - **VIX 40超**: 「パニックモード」として全ての新規買い判定をブロック。
  - **VIX 30〜40**: 「高リスク / 警戒モード」。判定は実行されますが、警告ラベル（VIX高リスク）が付与されます。
  - **VIX 25〜30**: 「警戒モード」として注意喚起。
  - **NK225 75日線下**: スイングモード時に「市場注意」ラベルを付与。

  **（追加）**
  - **米10年金利（US10Y） > 4.5%**: グロース株・ハイテク株を減点
  - **USD/JPY急騰（例: +2%以上/日）**: 日本の輸出株を加点
  - **NASDAQが75日線下**: グロース株全体を減点

- **高速化**: 判定時の遅延を防ぐため、上記指標はバックエンドで並列取得（Parallel Fetching）されます。

---

### Layer 2: ファンダメンタル（中長期）
- **対象**: 日本株（J-Quants / yfinance）、米国株（yfinance）。
- **評価項目**: ROE、営業利益成長率、PER（割安性）、PBR（資産性）。

**（追加: 日本株特有ロジック）**
- **PBR < 1**: 東証改革テーマ銘柄として加点
- **ROE > 15%**: 高収益企業として加点

- **デイトレモード**: このレイヤーは分析から**完全に除外**されます。

---

### Layer 3: テクニカル（短期〜中長期）
- **指標**:  
  EMA（5/20/75日）、RSI（相対力指数）、出来高。

**（追加指標）**
- MACD（トレンド転換）
- VWAP（機関投資家平均取得価格）
- Bollinger Bands（ボラティリティ）

- **スイング**:
  - 75日線の向き、ゴールデンクロス、RSI（30-60）を重視。
  - **（追加）MACDがシグナル上抜け時は加点**
  - **（追加）株価がVWAPより上の場合は加点**

- **デイトレ**:
  - **5日EMA**との乖離、短期ゴールデンクロスの勢いを重視。
  - **ATR (Average True Range)** によるボラティリティ判定（動かない銘柄は減点）。
  - **（追加）VWAPブレイク時は短期モメンタム加点**

---

### Layer 4: 定性・ニュース（AI判定）
- **AI**: Gemini AI による感情分析（0〜10点）。

- **スイング時プロンプト**: 中長期的な成長への影響を分析。

- **デイトレ時プロンプト**:
  - 本日から翌日にかけての「短期的な盛り上がり・材料視」を最優先で分析。
  - ボラティリティ（値幅）を伴うニュースかをスコアリング。

**（追加）**
- **ニュース頻度（news_count_24h）** を評価  
  - 急増している場合は短期トレンド発生として加点

---

## 2. 倍率とノーマライズ（正規化）

各モードによって解析項目が異なるため、合計値を最大スコアで割った「達成率」でシグナルを決定します。

- **シグナル基準**:
  - 85%以上: `Strong Buy`
  - 65%以上: `Buy`
  - 45%以上: `Hold`
  - それ以外: `Sell / Avoid`

---

## 3. 流動性とリスク管理

- **流動性チェック**:
  - デイトレモード時はスイングの5倍（日本株なら1日5億円以上）の売買代金を要求。不足時は警告を表示。

**（追加）**
- **時価総額チェック**
  - 時価総額 < 100億円の場合はボラティリティリスクとして警告

- **損切りラインの動的調整**:
  - スイング: 買値から -7%、高値から -10%。
  - デイトレ: 買値から -2%、高値から -3%。

- **エラー処理とフォールバック**:
  - 各API（J-Quants, Gemini）が失敗した場合は、自動的にフォールバック（yfinance, キーワード分析等）が行われ、利用可能なデータのみで最大スコアを再計算します。
  - ネットワーク遅延対策として、マクロ指標の取得には並列処理（Threading）を導入し、モバイル環境等でのタイムアウトを防止しています。
